# talk-to-minutes — 會議錄音 → 結構化報告

你（agent）是這個工具的操作員。使用者會丟一個會議錄音檔給你，你的工作是：
轉逐字稿 → 讓使用者選報告格式 → 依格式產出報告。全程用繁體中文，不用簡體。

## 標準流程（使用者給你音檔時）

1. **轉逐字稿**：執行
   ```bash
   ./transcribe.sh <音檔路徑>
   ```
   （英文會議加第二個參數 `English`。）完成後逐字稿在 `transcripts/<檔名>.txt`。
2. **讓使用者選格式**：讀 `formats/INDEX.md`，把格式清單用編號列給使用者選。
   使用者若已在訊息裡指明格式，直接用，不用再問。
3. **產出報告**：讀選定的 `formats/<格式檔>.md`，完整依照其中的「報告結構」
   與「整理原則」，從逐字稿整理出報告。
4. **存檔**：報告存到 `reports/YYYY-MM-DD-<簡短主題>.md`，並在對話中給出完整內容。
5. **更新預覽頁**：執行 `python3 build_viewer.py`。使用者說「打開 app／打開格式庫」
   時執行 `open app/index.html`。

## 設計新格式（使用者說「我要新格式」「這格式不合用」時）

兩種情況：

- **使用者描述情境**（例如「這是給業務主管看的季度客戶回顧」）→ 你根據情境
  設計標題結構與整理原則，先給使用者看草稿，確認後才存檔。
- **使用者直接指定標題與思考方式** → 照使用者的規格寫成格式檔，存檔。

存檔步驟：

1. 新格式寫入 `formats/<英文短名>.md`，結構比照現有格式檔
   （四段：適用情境／報告結構／整理原則／範例語氣，可增不可減）。
2. 在 `formats/INDEX.md` 表格**尾端**追加一列（編號遞增）。
3. 執行 `python3 build_viewer.py` 更新預覽頁。
4. 跟使用者確認已儲存，下次可直接點選。

修改既有格式：先把原檔內容備份到檔案尾端的 `<!-- 舊版 YYYY-MM-DD -->` 註解區塊，
再改正文。

## 整理報告的鐵則（所有格式共通）

- **不得編造**：報告中每一項內容都必須能對應到逐字稿；逐字稿沒有的不准寫。
- 數字、日期、金額、人名一律照逐字稿原文；聽不清或不確定的標「（待確認）」。
- Whisper 不分辨發言者。可從上下文合理推測（如「客戶方提到…」），
  但推測就要用推測的語氣，不得寫成斷言。
- 逐字稿若混有簡體字，報告一律轉成繁體。
- 報告開頭固定放 metadata：日期、音檔名、使用格式、逐字稿路徑。

## 環境需求（第一次在新機器上用）

- macOS + Homebrew，執行 `brew install ffmpeg openai-whisper`
- 第一次轉錄會自動下載 turbo 模型（約 1.5GB），之後離線可用
- 報告整理由你（正在讀這份文件的 agent）完成，不需要任何 API key
